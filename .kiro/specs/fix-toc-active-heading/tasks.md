# Implementation Plan

- [x] 1. useActiveHeading hook を scroll ベースの追跡に置き換える
- [x] 1.1 IntersectionObserver を廃止し scroll イベントリスナーで見出し位置を追跡する
  - IntersectionObserver、visibleHeadings Map、observer 変数を全て削除する
  - scroll イベントリスナーを `{ passive: true }` で登録し、`getBoundingClientRect` でコンテナ上端から各見出しの位置を判定する
  - コンテナ上端 + 40px のしきい値以下にある最後の見出しをアクティブとする
  - どの見出しもしきい値を超えていない場合、最初の見出しをフォールバックとして選択する
  - `requestAnimationFrame` で `ticking` フラグを使い、1フレームあたり最大1回の位置計算に制限する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2_

- [x] 1.2 ライフサイクル管理を scroll リスナー方式に合わせる
  - `disconnect()` で `cleanupFn`（scroll リスナー解除関数）を呼び出して null に戻す
  - `observe()` 先頭で `disconnect()` を呼び、既存リスナーを確実に解除してから新しいリスナーを登録する
  - `onCleanup(disconnect)` でコンポーネントアンマウント時の自動クリーンアップを維持する
  - 見出し要素が0件の場合は `activeId` を `null` に設定しリスナーを登録しない
  - _Requirements: 3.1, 3.2, 3.3_

- [x] 1.3 コンテンツ変更時に `observe()` 再呼出しで正しく追従することを確認する
  - `observe()` が呼ばれた直後に `update()` を即座に実行し、スクロール前でも現在位置に基づくアクティブ見出しを設定する
  - コンテンツ変更で DOM が入れ替わった場合、新しい見出し要素を `querySelectorAll` で再取得する
  - スクロール位置が保持されたままコンテンツが更新された場合でも、変更前と同じセクションの見出しがアクティブになることを確認する
  - _Requirements: 1.4, 4.1, 4.2_

- [ ] 2. 手動テストで全要件の動作を検証する
- [ ] 2.1 スクロール追跡の動作確認
  - 見出しを含む Markdown ファイルを開き、上から下へスクロールして各セクションで対応する見出しがハイライトされることを確認する
  - 下から上へスクロールして追従が正しいことを確認する
  - 最上部で最初の見出し、最下部で最後に通過した見出しがハイライトされることを確認する
  - ファイルウォッチでコンテンツが更新された後もハイライトが正しくリセットされることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 4.1, 4.2_
