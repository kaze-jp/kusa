# Implementation Plan

- [ ] 1. バッファ分割の状態管理ストアを作成する
- [ ] 1.1 分割状態を管理するストアを実装する
  - 分割の有効/無効、左右ペインのバッファID、フォーカスペイン、バッファピッカー表示フラグを管理するリアクティブストアを作成する
  - 分割を開始する操作を実装する。現在のアクティブタブIDを左ペインに設定し、右ペインを未選択状態にして、バッファピッカーを表示する
  - 分割を終了する操作を実装する。全状態をリセットして初期状態に戻す
  - フォーカスペインの切り替え操作を実装する
  - 指定されたタブIDでフォーカスペインのバッファを切り替える操作を実装する
  - タブリストを受け取り、フォーカスペインのバッファを次/前に巡回する操作を実装する
  - _Requirements: 1.1, 1.3, 1.4, 2.1, 2.2, 3.1, 3.2, 4.2_

- [ ] 1.2 ストアのユニットテストを作成する
  - 分割開始時に左ペインにタブIDが設定され右ペインが未選択であることを検証する
  - 分割終了で全状態がリセットされることを検証する
  - フォーカスペイン切り替えが正しく動作することを検証する
  - バッファ巡回が末尾から先頭にラップすることを検証する
  - _Requirements: 1.1, 1.4, 2.1, 2.2, 3.2_

- [ ] 2. ペイン表示用のUIコンポーネントを作成する
- [ ] 2.1 (P) 各ペインのコンテナコンポーネントを実装する
  - ペイン上部にファイル名を表示するヘッダー領域を作成する
  - フォーカスされたペインには上部に青いボーダーのアクティブインジケーターを表示する
  - ペインをクリックしたときにフォーカスが移動するようにする
  - ヘッダーの下にMarkdownプレビューを表示する。既存のPreviewコンポーネントを利用し、各ペインが独立したスクロール位置を持つ
  - _Requirements: 1.3, 2.3, 2.4, 3.3, 5.4_

- [ ] 2.2 (P) バッファ選択ピッカーコンポーネントを実装する
  - 現在開いているタブの一覧をリスト表示する
  - j/kキーで選択行を上下に移動し、Enterで確定、Escでキャンセルできるようにする
  - 選択されたバッファのタブIDをコールバックで通知する
  - ペインの右側中央に重ねて表示するオーバーレイスタイルにする
  - _Requirements: 3.1_

- [ ] 3. Ctrl-W 2ストロークキーバインドを実装する
- [ ] 3.1 Ctrl-W プレフィックスの入力検知を実装する
  - Ctrl-W を押したときに待機フラグを立て、500msのタイムアウトを設定する
  - タイムアウト内に次のキーが押された場合、対応する操作を実行して待機フラグをクリアする
  - タイムアウトを超えた場合は待機フラグを自動クリアする（何もしない）
  - _Requirements: 1.1, 1.4, 2.1, 2.2_

- [ ] 3.2 各キーに対応するアクションをマッピングする
  - `v`: プレビューモード時に分割を開始する。editモードやバッファ分割中は無視する
  - `q`: バッファ分割中に分割を終了する
  - `h`: バッファ分割中に左ペインにフォーカスを移動する
  - `l`: バッファ分割中に右ペインにフォーカスを移動する
  - _Requirements: 1.1, 1.4, 2.1, 2.2, 4.3_

- [ ] 3.3 バッファ分割中のShift+H/Lでペイン内バッファ切替を実装する
  - フォーカスされたペインで次/前のバッファに切り替える
  - 既存のShift+H/L（タブ移動）の動作をバッファ分割中のみオーバーライドする
  - _Requirements: 3.2_

- [ ] 4. App.tsx にバッファ分割表示を統合する
- [ ] 4.1 バッファ分割時のレンダリング分岐を追加する
  - バッファ分割がアクティブな場合、既存の単一プレビュー表示の代わりに SplitLayout を使い左右にペインコンポーネントを配置する
  - 各ペインに対応するタブの内容をMarkdownパースしてHTMLとして渡す
  - バッファピッカーが表示中の場合、右ペイン上にピッカーを重ねて表示する
  - _Requirements: 1.1, 1.2, 5.1, 5.2, 5.3, 5.4_

- [ ] 4.2 既存モードとの排他制御を実装する
  - バッファ分割中は `Ctrl+E`（エディタモード切替）を無効にするガード条件を追加する
  - エディタモードおよびSplit mode中は `Ctrl-W v`（バッファ分割開始）を無効にするガード条件を追加する
  - バッファ分割終了時に元のプレビューモードに正しく復帰することを確認する
  - _Requirements: 4.1, 4.2, 4.3_

- [ ] 4.3 タブバーのハイライトをバッファ分割に対応させる
  - バッファ分割中はフォーカスされたペインのバッファに対応するタブをハイライトする
  - フォーカスペインが変わったときにハイライトが追従することを確認する
  - _Requirements: 3.4_

- [ ] 4.4 タブ削除時のクリーンアップを実装する
  - ペインに割り当てられたタブが閉じられた場合、バッファ分割を自動的に終了して単一プレビューに戻る
  - _Requirements: 1.4, 4.2_
