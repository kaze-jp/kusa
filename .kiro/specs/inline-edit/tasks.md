# Implementation Plan

## Tasks

- [ ] 1. Rust バックエンド: ファイル書き込み IPC コマンド
- [ ] 1.1 ファイル書き込みコマンドを実装する
  - 指定パスにテキスト内容を UTF-8 で書き込むコマンドを作成
  - アトミック書き込み（一時ファイル作成→リネーム）でクラッシュ安全性を確保
  - パストラバーサル防止のためパスを正規化し、許可範囲を検証
  - 書き込みバイト数を含む構造化レスポンスを返す
  - エラー時（権限不足、ディスク容量不足、パス不存在）は構造化エラーを返す
  - Tauri の fs 権限スコープに書き込みアクセスを追加
  - Rust ユニットテストで正常系・異常系を検証
  - _Requirements: 4.1, 4.2, 4.3, 4.5_

- [ ] 2. CodeMirror エディタ基盤
- [ ] 2.1 CodeMirror モジュールの遅延ロード機構を構築する
  - CodeMirror 関連パッケージ（@codemirror/view, @codemirror/state, @codemirror/lang-markdown, @codemirror/theme-one-dark, @replit/codemirror-vim）をインストール
  - Vite の dynamic import で CodeMirror モジュール群を一括ロードするサービスを作成
  - ロード状態（idle / loading / loaded / error）を Signal で管理
  - ロード完了後はキャッシュを保持し再ロード不要にする
  - ロード中のインジケーターコンポーネントを作成
  - Preview モード起動時に CodeMirror がロードされないことを確認
  - _Requirements: 6.1, 6.2, 6.3, 6.4_

- [ ] 2.2 CodeMirror エディタの初期化・管理サービスを構築する
  - CodeMirror EditorView を指定 DOM 要素にマウントするサービスを作成
  - 拡張の組み立て: Markdown シンタックスハイライト、行番号、ダークテーマ
  - ファイル内容の初期セットと外部からのコンテンツ更新
  - コンテンツ変更リスナーの登録とコールバック通知
  - カーソル位置変更のリスナーとコールバック通知
  - EditorView の destroy によるクリーンアップ
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5_

- [ ] 2.3 Vim extension を統合しカスタムコマンドを登録する
  - @replit/codemirror-vim を CodeMirror の拡張として組み込む
  - jk escape がデフォルトで動作することを確認
  - Vim.defineEx で `:w`, `:wq`, `:q` のカスタムExコマンドを登録
  - `:w` で保存コールバック、`:wq` で保存+終了コールバック、`:q` でウィンドウクローズを実行
  - Vim モード変更イベント（NORMAL / INSERT / VISUAL / COMMAND）をコールバックで通知
  - Vim motions（w, b, e, dd, yy, p, x, cw 等）の基本動作を検証
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7, 4.1, 4.2_

- [ ] 3. Split View レイアウト
- [ ] 3.1 リサイズ可能な Split Layout コンポーネントを作成する
  - CSS Flexbox で左ペイン + ディバイダー + 右ペインの横並びレイアウトを構築
  - ディバイダーの PointerEvent ハンドリングでペイン幅のドラッグリサイズを実装
  - ペイン幅比率を Signal で管理し、デフォルトを 50:50 に設定
  - 最小ペイン幅の制約（各ペイン最低 200px）を適用
  - ウィンドウリサイズ時にペイン比率を維持するロジック
  - ダークテーマに合わせたディバイダーのスタイリング
  - _Requirements: 1.1, 1.2, 1.4, 1.5_

- [ ] 3.2 (P) エディタペインコンポーネントを作成する
  - CodeMirror EditorView をマウントする SolidJS コンポーネントを作成
  - LazyLoader からロード済みモジュールを受け取り、CMEditor サービスで初期化
  - コンポーネントの onCleanup で EditorView を destroy
  - ペインサイズ変更時に EditorView のリサイズを通知
  - _Requirements: 2.1_

- [ ] 3.3 (P) ステータスバーコンポーネントを作成する
  - Vim モード表示（NORMAL / INSERT / VISUAL）を左側に配置
  - カーソル位置（行:列）を右側に配置
  - 保存状態通知（成功/エラー）を中央に配置し 2 秒後にフェードアウト
  - 未保存インジケーター（ドットまたはアイコン）を表示
  - ダークテーマに合わせたスタイリング
  - _Requirements: 2.5, 3.6, 4.4, 4.5, 4.6_

- [ ] 4. リアルタイム同期と自動保存
- [ ] 4.1 エディタ→プレビュー同期エンジンを構築する
  - エディタのコンテンツ変更を受け取りデバウンス（200-300ms）するサービスを作成
  - デバウンス後に既存 MarkdownPipeline を呼び出し HTML を生成
  - 生成された HTML を Preview コンポーネントに反映する仕組みを接続
  - プレビュー更新時に前回の表示を維持してちらつきを防止
  - Markdown パース失敗時は最終成功プレビューを維持する
  - _Requirements: 1.3, 7.1, 7.2, 7.3_

- [ ] 4.2 デバウンス付き自動保存ハンドラーを構築する
  - エディタの変更から 500ms-1s 後に write_file IPC を呼び出す auto-save ロジック
  - 保存中は追加の保存要求をキューイングし、保存完了後に最新内容で再保存
  - 保存完了時に isDirty フラグを false に更新
  - 保存失敗時はエラーをコールバックで通知し、isDirty を true に維持
  - 未保存変更の有無を追跡する isDirty Signal を管理
  - _Requirements: 4.3, 4.6_

- [ ] 4.3 Vim の :w コマンドから手動保存を接続する
  - Vim の `:w` コマンドで write_file IPC を即座に呼び出す接続
  - `:wq` コマンドで保存完了後にウィンドウを閉じる処理
  - 保存結果（成功/失敗）を StatusBar に通知する接続
  - auto-save のデバウンスタイマーをリセット
  - _Requirements: 4.1, 4.2, 4.4, 4.5_

- [ ] 5. 表示モード切替
- [ ] 5.1 App シェルの viewMode を拡張し3モードに対応する
  - viewMode Signal を 'preview' | 'edit' | 'split' | 'file-list' | 'error' に拡張
  - モードに応じたコンポーネント表示の条件分岐（Show / Switch）
  - Preview モードの表示は既存のままを維持
  - Edit モードではエディタのみ表示、Split モードでは SplitLayout を表示
  - デフォルトモードを Preview に設定
  - _Requirements: 5.1, 5.3_

- [ ] 5.2 キーボード操作によるモード切替ロジックを実装する
  - Preview モードで Enter または i を押すと Split モードに遷移しエディタにフォーカス
  - Escape 2回連続の検知ロジック（タイムアウト付き状態管理）
  - Escape 2回で Preview モードに遷移
  - CodeMirror がフォーカスされている場合は Enter / i のモード切替を無視（Vim 操作優先）
  - 初回 Split/Edit 切替時に LazyLoader を呼び出し CodeMirror をロード
  - ロード完了後にエディタを初期化しフォーカスを移す
  - _Requirements: 5.2, 5.4, 5.5_

- [ ] 6. 統合とテスト
- [ ] 6.1 全コンポーネントをアプリケーションシェルに統合する
  - SplitLayout に EditorPane と Preview を子要素として接続
  - SyncEngine をエディタとプレビューの間に配線
  - AutoSaveHandler を SyncEngine と StatusBar に接続
  - VimExtension のコールバック（save, quit, mode change）を各ハンドラーに接続
  - StatusBar を App シェルの下部に配置しモード・カーソル・保存状態を表示
  - ViewModeSwitch のイベントを App の viewMode Signal に接続
  - _Requirements: 1.1, 1.3, 2.1, 3.1, 4.1, 5.1_

- [ ] 6.2 全フローの結合テストを実施する
  - CLI 起動 → Preview 表示 → i 押下 → Split view 表示 → エディタにファイル内容表示
  - Split view でテキスト編集 → プレビューにリアルタイム反映
  - `:w` コマンド → ファイル保存 → StatusBar に保存通知
  - `:wq` コマンド → ファイル保存 → ウィンドウクローズ
  - auto-save → 一定時間後にファイル保存
  - Escape 2回 → Preview モード復帰
  - CodeMirror 遅延ロード → ロード中インジケーター → 完了後エディタ表示
  - _Requirements: 1.1, 1.3, 4.1, 4.2, 4.3, 5.4, 5.5, 6.2, 7.1_

- [ ] 6.3 エラーケースとエッジケースのテストを実施する
  - 読み取り専用ファイルへの保存試行 → エラー表示、編集内容保持
  - 不正な Markdown 編集 → プレビューが最終成功状態を維持
  - CodeMirror ロード失敗 → リトライ可能なエラー表示
  - 高速タイピング時のデバウンス動作 → 過剰なパース/保存が発生しないことを確認
  - ウィンドウリサイズ → Split view のペイン比率維持
  - _Requirements: 4.5, 7.3, 6.3_

## Parallel Execution Notes

タスク1（Rust バックエンド）とタスク2（CodeMirror 基盤）は互いに独立のため並列実行可能。

タスク2内部:
- **Task 2.1**: 他のタスクの前提条件（パッケージインストール + ロード機構）
- **Task 2.2 + 2.3**: 2.1 完了後に並列実行可能（CMEditor と VimExtension は独立して構築可能だが、2.3 は 2.2 の EditorView 上に統合されるため実質的に順序依存）

タスク3内部:
- **Task 3.2 + 3.3**: 3.1 完了後に並列実行可能（EditorPane と StatusBar は互いに独立）

タスク4: タスク2 と 3 の完了後に実行（エディタとレイアウトに依存）
タスク5: タスク4 と既存 Preview/App の完了後に実行
タスク6: 全タスク完了後に統合・テスト実行
